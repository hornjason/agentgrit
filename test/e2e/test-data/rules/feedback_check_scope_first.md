Read the scope boundary in the issue or PRD before writing any code.

- **Why:** Implementing outside scope wastes time and creates merge conflicts with other workstreams
- **How to apply:** Before touching any file, verify it falls within the declared scope boundary. If unclear, ask before proceeding.
